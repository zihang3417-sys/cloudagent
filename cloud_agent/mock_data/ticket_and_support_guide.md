# 云平台智能客服与工单运维指南

## 1. 概述
为保障云平台用户的业务连续性，平台提供多层级的客户服务与运维技术支持体系。当用户遇到资源使用、计费疑问或底层系统故障时，可通过提交工单或联系智能客服获取协助。

## 2. 客户服务分级与响应 SLA

### 2.1 智能客服 (L1/L2)
*   **响应时间**：7x24 小时实时响应。
*   **适用场景**：
    *   **L1 (语义缓存直返)**：解决高频标准化问题。例如：“如何查询实例详情？”、“包年包月退款手续费是多少？”、“ecs.g8a.xlarge 的带宽上限是多少？”。
    *   **L2 (大模型检索增强)**：解决多条件约束的复杂问题。例如：“北京可用区 K 支持哪些带 GPU 的抢占式实例且能装 Windows？”、“我的 ECS 连不上 RDS，需要排查哪些网络和安全组配置？”。
*   **升级机制**：如果智能客服判断问题置信度极低（低于 0.4），或者用户连续两次反馈“未解决”，系统将自动流转至人工兜底。

### 2.2 人工工单支持 (Ticket Support)
*   **支持级别**：
    *   **基础服务 (Basic)**：工作日 9:00-18:00 响应，一般在 4-8 小时内首次回复。适用于常规咨询。
    *   **企业级支持 (Enterprise)**：7x24 小时响应，P1 级别故障（如核心数据库宕机）在 15 分钟内响应。
*   **工单提交建议**：提交工单时，强烈建议提供完整的**实例 ID (如 i-bp1xxx)**、**地域 (Region)**、**报错信息截图**或**具体的错误码 (如 InvalidInstanceType.NotFound)**，以便工程师快速定位。

## 3. 常见运维工单分类与排查

### 3.1 资源变配与扩容失败
*   **工单类型**：计算与网络 > 实例规格变配。
*   **常见问题**：
    *   用户尝试将 `ecs.c7.large` 升级为 `ecs.g8a.xlarge`，但控制台提示“库存不足”或“不支持的变配路径”。
    *   用户尝试为实例挂载第三块云盘，但系统提示 `OperationDenied.StorageNotSupported`。
*   **运维建议**：建议用户先通过智能客服查询变配矩阵，或检查目标可用区是否有足够的底层物理机资源。

### 3.2 网络不通与延迟高
*   **工单类型**：网络与安全 > 专有网络 VPC / 弹性公网 IP。
*   **常见问题**：
    *   实例无法访问外网，或者外部无法通过 SSH 登录。
    *   两个不同地域的 VPC 之间无法内网互通。
*   **运维建议**：此类问题 90% 是由**安全组配置错误**或**未绑定 EIP**导致的。工程师通常会先要求用户提供安全组的入方向规则截图，并确认 VPC 路由表中是否存在目标网段的路由条目。

### 3.3 账单与费用争议
*   **工单类型**：财务与账单 > 退订与退款。
*   **常见问题**：
    *   用户申请“五天无理由退款”被拒绝。
    *   按量付费实例已关机，但账户仍然在持续扣费。
*   **运维建议**：客服会核对用户的退款额度是否超限（每个账号每年限 1 次），并解释“停机不收费”功能的限制条件（如本地盘实例不支持）。

## 4. 底层系统维护与自动演练
*   **系统事件 (System Events)**：云平台会定期对底层物理机（宿主机）进行升级、补丁修复或硬件替换。
*   **用户感知**：如果用户的实例（如 RDS 主节点）所在的物理机命中维护计划，系统会提前 3-7 天发送“实例将发生重启或迁移”的通知。
*   **运维建议**：强烈建议企业用户采用**高可用版 RDS (主备双节点)** 和跨可用区部署的 ECS 集群，以实现底层维护对上层业务的透明无感。

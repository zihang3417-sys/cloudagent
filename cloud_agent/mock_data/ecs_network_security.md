# 云平台网络规划与安全控制策略 (VPC / 安全组 / EIP)

## 1. 专有网络 VPC 与子网划分
企业在云上构建架构时，必须先规划 VPC。VPC (Virtual Private Cloud) 是地域级别的逻辑隔离网络。

### 1.1 核心组件与拓扑
*   **VPC 实例**：如 `vpc-prod-beijing`（北京生产环境）。网段通常选择 RFC1918 私网地址，如 `10.0.0.0/8`。
*   **交换机 (VSwitch)**：VPC 下必须划分交换机，**交换机是可用区级别的**。
    *   **vsw-beijing-k**：属于 `vpc-prod-beijing`，部署在可用区 K (`cn-beijing-k`)。建议将核心数据库 (如 RDS) 和企业级实例 (如 ecs.g8a.4xlarge) 部署在此处。
    *   **vsw-beijing-g**：属于 `vpc-prod-beijing`，部署在可用区 G (`cn-beijing-g`)。注意：此交换机下无法部署 GPU 实例 (gn7i)。

### 1.2 内网互通与隔离机制
*   **同 VPC 互通**：部署在 `vsw-beijing-k` 和 `vsw-beijing-g` 下的实例（即使在不同可用区），内网是默认完全互通的（不受网段限制，但受安全组控制）。
*   **跨 VPC 隔离**：如果开发环境使用了 `vpc-dev-beijing`，那么生产环境 `vpc-prod-beijing` 的 ECS 默认绝对无法访问开发环境。必须配置 **云企业网 (CEN)** 实现路由打通。

## 2. 弹性公网 IP (EIP) 与共享带宽
ECS 实例需要公网访问能力时，建议绑定 EIP。

### 2.1 绑定与解绑限制
*   **绑定目标**：EIP 只能绑定在**专有网络类型**的资源上。例如，可以绑定在 `ecs.c7.large` 的主弹性网卡 (Primary ENI) 或辅助弹性网卡 (Secondary ENI) 上。
*   **区域约束**：EIP 是地域级资源。北京的 EIP 只能绑定给北京地域的 ECS，**严禁跨地域绑定**。

### 2.2 共享带宽 (Shared Bandwidth)
*   **场景**：如果有多台 ECS (如 10 台 `ecs.g8a.xlarge` Web 服务器) 均绑定了 EIP，建议将这些 EIP 移入同一个共享带宽实例。
*   **优势**：可以应对突发流量峰值，同时降低 30% - 40% 的公网带宽成本。**注意**：包年包月的 ECS 实例如果使用的是“按固定带宽”计费，无法加入共享带宽，必须先变更为“按使用流量”计费。

## 3. 安全组 (Security Group) 与弹性网卡 (ENI)
安全组是基于有状态数据包检测的虚拟防火墙，控制实例的进出流量。

### 3.1 弹性网卡 (ENI) 与安全组的关联
安全组的生效粒度是**弹性网卡 (ENI)**，而不是直接作用于 ECS 实例。
*   **多安全组策略**：一块 ENI（如 ecs.g8a.4xlarge 绑定的第 2 块网卡）最多可加入 **5 个安全组**。系统会取这些安全组规则的并集。
*   **多网卡隔离**：如果 `ecs.c7.8xlarge` 绑定了 2 块网卡。您可以让网卡 A 加入 `sg-web`（开放 80/443），网卡 B 加入 `sg-db`（仅内网互通，禁止公网），实现内外网流量物理隔离。

### 3.2 典型企业级安全组配置
*   **sg-web-prod** (Web 集群安全组)
    *   **入方向**：允许 `0.0.0.0/0` 访问 TCP `80`, `443`。允许特定堡垒机 IP (如 `203.0.113.1/32`) 访问 TCP `22` (SSH)。拒绝其他所有入站流量。
    *   **出方向**：允许所有出站流量。
*   **sg-rds-internal** (数据库安全组)
    *   **入方向**：**拒绝所有公网访问**。配置授权策略：仅允许源安全组为 `sg-web-prod` 的实例，通过内网访问 TCP `3306` 端口。这是一种极高安全性的**基于安全组 ID 授权**的配置。

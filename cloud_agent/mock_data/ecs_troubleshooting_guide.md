# 云服务器 ECS 高级故障排查与规格级错误码速查

## 1. 概述
本手册针对企业级云服务器 ECS（特别是 g8a、c7、gn7i 等规格族）在底层硬件调度、网络网卡挂载及系统启动时的深度故障排查。

## 2. 实例生命周期状态与底层调度错误码
当调用 API `RunInstances` 或在控制台发起启动、变配操作时，底层的神龙调度器或存储控制器可能会返回以下特定状态码。

### 2.1 OperationDenied.StorageNotSupported
*   **触发场景**：为实例挂载不兼容的块存储（云盘）。
*   **关联规格约束**：
    *   如果您尝试将 **本地 NVMe SSD** 挂载给 **ecs.g8a.xlarge** 或 **ecs.c7.large** 实例，会触发此报错。因为 g8a 和 c7 系列基于云盘架构，**不支持**本地盘。
    *   如果您尝试用 **ESSD PL0 云盘** 启动 **ecs.gn7i-c8g1.2xlarge (GPU 实例)**，也会触发此报错。gn7i 强制要求搭配 **ESSD PL2 及以上**级别的系统盘启动，因为深度学习推理需要极高的系统盘 IOPS。
*   **解决方案 (Resolution)**：检查实例规格与块存储的兼容性矩阵，更换目标系统盘或数据盘的类型为 ESSD PL1 或 PL2。

### 2.2 InvalidInstanceType.ZoneMismatch
*   **触发场景**：创建实例时指定的可用区与实例规格的物理部署不匹配。
*   **关联区域约束**：
    *   在 **华北2（北京）可用区 G (cn-beijing-g)** 申请 **GPU 计算型 gn7i**。该可用区未部署 GPU 物理机架。
    *   在 **华北2（北京）可用区 K (cn-beijing-k)** 申请 **抢占式实例 (Spot Instance)** 时，如果该规格当前在 K 区库存耗尽，也会降级报出此错误。
*   **解决方案 (Resolution)**：建议 GPU 实例迁移至 **可用区 K** 部署。抢占式实例建议开启“可用区随机分配”策略，或选择 **可用区 L**（支持抢占式实例优先调度）。

### 2.3 InvalidNetworkInterface.ExceedQuota
*   **触发场景**：尝试为 ECS 实例绑定过多的弹性网卡 (ENI)。
*   **关联规格约束**：
    *   **ecs.c7.large**（2核4G）最多仅支持绑定 **2 个 ENI**（1个主网卡 + 1个辅助网卡）。如果尝试绑定第 3 个，将报错。
    *   **ecs.g8a.xlarge**（4核16G）最多支持绑定 **3 个 ENI**。
    *   **ecs.c7.8xlarge**（32核64G）最多支持绑定 **15 个 ENI**。
*   **解决方案 (Resolution)**：调用 `DescribeInstanceTypes` 接口查询当前规格的 `EniQuantity` 配额。如需更多网卡，必须升级（Upgrade）实例规格至更高核数。

## 3. 操作系统启动与连接深度排查
针对企业级镜像（Alibaba Cloud Linux 3 / Windows Server 2022）的启动级故障。

### 3.1 实例状态一直处于 Starting (启动中卡死)
*   **现象**：控制台显示实例一直为 Starting，无法进入 Running 状态。
*   **关联系统约束**：
    *   **Windows License 限制**：如果您在创建时选择了 **Windows Server 2022 数据中心版**，但实例规格选择了 **ecs.c7.large (2 vCPU)**，会导致系统启动失败。Windows Server 2022 **强制要求 4 vCPU 及以上** 规格（如 ecs.g8a.xlarge）。
    *   **解决方案**：强制停止实例，通过控制台“更换操作系统”变更为 Linux 3，或将实例规格升级至 4 vCPU 以上。

### 3.2 高并发网络丢包与 PPS 瓶颈排查
*   **现象**：Web 服务器在高并发时，出现 TCP 连接拒绝或大量丢包，但 CPU 和内存监控未满载。
*   **排查步骤与规格瓶颈**：
    1.  **检查网络包收发率 (PPS) 瓶颈**：不同规格的实例有严格的 PPS 物理上限。
        *   **ecs.c7.large** 的最大收发包能力为 **50 万 PPS**。
        *   **ecs.g8a.xlarge** 的最大收发包能力为 **200 万 PPS**。
    2.  **检查带宽上限 (Gbps)**：
        *   **ecs.c7.large** 的内/外网最大总带宽为 **5 Gbps**。如果突发流量超过 5 Gbps，底层虚拟化交换机会主动丢弃（Drop）数据包。
*   **解决方案**：如果在云监控中发现 PPS 或带宽触及上述阈值（如达到 50万 PPS），说明遇到了规格的物理天花板。此时调整安全组或系统参数无效，**必须变配**（例如升级到 ecs.c7.8xlarge，支持 1200 万 PPS 和 40 Gbps 带宽）。
